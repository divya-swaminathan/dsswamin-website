This site is built with **[Eleventy](https://www.11ty.dev/)**: posts are plain **Markdown** files in `src/posts/`. When you push to `main`, GitHub Actions runs `npm run build` and publishes the generated HTML to Pages.

### Add a new post

1. Create a file in `src/posts/` named `YYYY-MM-DD-short-slug.md` (the slug becomes the URL path under `/blog/`).
2. Start with YAML front matter (`title`, `description`, `date`), then your Markdown body. See the **README** in the repo for a copy-paste template.

3. Commit and push; the **Blog** index and **Atom feed** update automatically.

### Tips

- Keep employer-confidential details out; write for a public audience.
- Short posts are fine — one clear idea beats a long essay you never ship.
